## 1.2 IMU白噪声和偏置标定

但是上述的官方标定还是没有给出有关IMU在之后手眼标定或者是IMU和相机标定的

accelerometer_noise_density: #continous

accelerometer_random_walk:

gyroscope_noise_density: #continous

gyroscope_random_walk:

这几个参数.经过一番搜寻,发现还是只能用**[code_utils](https://link.zhihu.com/?target=https%3A//github.com/gaowenliang/code_utils)**和**[imu_utils](https://link.zhihu.com/?target=https%3A//github.com/gaowenliang/imu_utils)**进行标定,得到IMU的白噪声和偏置.

先录制imu的bag包,之后进行标定.录制的时候保持静止不动,因为标定的是IMU的白噪声和偏置.

## 3.6 将Kalibr外参转换为ORB-SLAM3的 `IMU.T_b_c1`

Kalibr联合标定结束后，一般会在 `data_cam_imu` 目录下生成类似下面的文件：

```bash
imu_cam-camchain-imucam.yaml
imu_cam-report-imucam.pdf
imu_cam-results-imucam.txt
```

其中 `imu_cam-camchain-imucam.yaml` 中通常会有如下字段：

```yaml
cam0:
  T_cam_imu:
    - [r11, r12, r13, tx]
    - [r21, r22, r23, ty]
    - [r31, r32, r33, tz]
    - [0.0, 0.0, 0.0, 1.0]
```

需要注意：Kalibr输出的 `T_cam_imu` 一般表示 **IMU坐标系到相机坐标系** 的变换，即：

```text
T_cam_imu = T_c_i
```

而ORB-SLAM3配置文件中的 `IMU.T_b_c1` 表示 **相机坐标系到IMU/body坐标系** 的变换，即：

```text
IMU.T_b_c1 = T_b_c1 = T_imu_cam
```

因此，如果直接使用Kalibr输出的 `T_cam_imu`，需要先对矩阵求逆：

```text
T_b_c1 = inverse(T_cam_imu)
```

设Kalibr输出为：

```text
T_cam_imu = [ R  t ]
            [ 0  1 ]
```

则转换后的ORB-SLAM3外参为：

```text
T_b_c1 = T_imu_cam = [ R^T   -R^T t ]
                       [  0       1   ]
```

可以在 `data_cam_imu` 目录下新建一个转换脚本：

```bash
cd ~/kalibr_ws/src/Kalibr/data_cam_imu
gedit convert_kalibr_to_orb.py
```

写入以下内容：

```python
#!/usr/bin/env python3
import yaml
import numpy as np

# 如果你的联合标定输出文件名不同，这里需要对应修改
kalibr_yaml = "imu_cam-camchain-imucam.yaml"

with open(kalibr_yaml, "r") as f:
    data = yaml.safe_load(f)

# 单目或第一个相机一般是 cam0
T_cam_imu = np.array(data["cam0"]["T_cam_imu"], dtype=float)

# ORB-SLAM3需要的是 T_b_c1，即 T_imu_cam
T_imu_cam = np.linalg.inv(T_cam_imu)

np.set_printoptions(precision=12, suppress=False)

print("Kalibr输出的 T_cam_imu 为：")
print(T_cam_imu)

print("\n转换后的 ORB-SLAM3 IMU.T_b_c1 为：")
print(T_imu_cam)

print("\n可以复制到ORB-SLAM3的yaml配置文件中：")
print("IMU.T_b_c1: !!opencv-matrix")
print("   rows: 4")
print("   cols: 4")
print("   dt: f")
print("   data: [" + ", ".join(f"{x:.12g}" for x in T_imu_cam.reshape(-1)) + "]")
```

运行脚本：

```bash
python3 convert_kalibr_to_orb.py
```

输出结果中 `IMU.T_b_c1` 部分可以直接复制到ORB-SLAM3配置文件中，例如：

```yaml
IMU.T_b_c1: !!opencv-matrix
   rows: 4
   cols: 4
   dt: f
   data: [r11, r12, r13, tx,
          r21, r22, r23, ty,
          r31, r32, r33, tz,
          0.0, 0.0, 0.0, 1.0]
```

注意事项：

1. 如果使用的是D435i彩色图像 `/camera/color/image_raw` 做联合标定，那么这里的 `c1` 就对应彩色相机坐标系。
2. ORB-SLAM3中的 `T_b_c1` 是从相机到IMU/body的变换，不要把Kalibr的 `T_cam_imu` 直接复制进去。
3. 如果外参方向填反，常见现象是IMU初始化失败、轨迹发散、频繁lost/reset，或者尺度和姿态明显异常。
4. 转换后可以检查平移量是否在合理范围内。D435i相机和IMU之间的平移通常是厘米量级，如果出现特别大的数值，需要检查标定过程、话题、时间同步和矩阵方向。
